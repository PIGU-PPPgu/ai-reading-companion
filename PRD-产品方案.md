# AI阅读家教 — 产品方案 (PRD v0.1)

> 让每个孩子都有一个专属的阅读导师

---

## 一、产品定位

**一句话**：AI带着中小学生精读一本书，拆解知识、生成图谱、互动讨论、家长可见。

**不是什么**：
- 不是电子书阅读器
- 不是AI摘要工具
- 不是题库/练习册

**是什么**：一个 **阅读家教** — 你请了一个大学生家教陪孩子读书，但这个家教是AI，24小时在线，只要几十块/月。

---

## 二、目标用户

| 角色 | 需求 | 痛点 |
|------|------|------|
| **学生**（7-15岁） | 读名著很无聊/读不懂 | 没人引导，读完就忘 |
| **家长** | 想让孩子多读书但没时间陪 | 不知道孩子读懂了没，不知道怎么讨论 |

**切入点**：中考必读名著（12本） → 扩展到课标推荐书目 → 最终覆盖所有青少年读物

---

## 三、核心功能

### 3.1 书籍拆解引擎
- **输入**：书籍PDF/EPUB/文本
- **输出**：
  - 章节结构 + 核心知识点
  - 人物图谱（关系网络）
  - 主题图谱（概念关系）
  - 中考考点标注（如果是必读名著）
  - 难度评估 + 适配阅读计划

### 3.2 个性化带读
- **阅读计划生成**：根据学生年级、阅读速度、时间安排自动生成
- **每日推送**：今天读什么 + 重点提示 + 讨论题
- **自适应节奏**：学生读得快就加速，理解弱就多讨论
- **家长可调控**：家长可以调速度、设目标、要求加练

### 3.3 互动讨论（核心体验）
- **苏格拉底式提问**：AI不直接给答案，通过提问引导思考
- **理解检测**：每章后自动出题，检测理解深度
- **笔记系统**：学生的想法可以加到知识图谱中
- **语音对话**：支持语音交互（像和家教聊天）

### 3.4 知识图谱（Textbook Graph理念）
- **动态生长**：随着阅读推进，图谱逐步展开
- **可视化**：学生可以看到自己的知识网络
- **跨书关联**：读完多本书后，图谱之间产生连接
- **个性化标记**：学生标注的重点、疑问都反映在图谱中

### 3.5 音频伴侣（Aibrary灵感）
- **每日音频摘要**：AI生成3-5分钟播客式回顾
- **故事化讲述**：不是机械摘要，而是讲故事的风格
- **家长周报音频**：10分钟总结，告诉家长孩子读了什么
- **个性化**：根据学生的理解程度调整讲解深度

### 3.6 家长端
- **阅读进度看板**：读了多少、花了多长时间
- **理解度报告**：哪些掌握好、哪些需要关注
- **讨论指引**："今晚你可以和孩子聊..."
- **周报/月报**：阅读习惯养成追踪

### 技术选型
- **LLM**：GLM-5 / Claude（对话+拆解）
- **向量数据库**：Milvus / Qdrant（书籍RAG检索）
- **图数据库**：Neo4j（知识图谱存储和查询）
- **TTS**：硅基流动 / Fish Audio（音频生成）
- **前端**：微信小程序（MVP）→ 后续H5/App
- **后端**：Node.js + FastAPI

## 五、商业模式

### 收费模式
- **免费**：1本书体验（选一本试读）
- **月卡**：¥39/月（无限阅读）
- **年卡**：¥299/年
- **单本购买**：¥9.9/本

### 成本估算（月/用户）
- LLM调用：¥5-8（对话量适中）
- TTS生成：¥2-3
- 服务器/存储：¥2
- **总成本约 ¥10/用户/月**
- **毛利率 70%+**

### 增长策略
1. **中考名著切入**（刚需，家长愿意付费）
2. **家长社群裂变**（周报可分享，"我家孩子这周读了..."）
3. **学校合作**（B端，阅读课/课后服务）
4. **内容扩展**（从名著到科普、历史、英语原版）

## 八、关键风险

1. **内容质量**：AI拆书可能不准确 → 必读名著由老师人工审核
2. **学生注意力**：孩子会不会认真读 → 游戏化+讨论互动
3. **家长付费意愿** → 免费试读1本 + 中考刚需驱动
4. **版权问题** → 不提供书籍全文，只做引导+讨论
