## 一、核心思路：缝合怪策略

### 为什么是缝合怪？

市场上已经有很好的开源组件，我们不需要从零搭建。找到最合适的基座，缝上我们的核心价值——**K12教学设计 + 结构化带读 + 家长闭环**。

### 缝合清单

| 组件 | 来源 | 用途 | 许可证 |
|------|------|------|--------|
| **基座平台** | [DeepTutor](https://github.com/HKUDS/DeepTutor)（港大） | Agent架构、RAG、知识图谱、对话管理、插件体系、Docker部署 | Apache-2.0 ✅ |
| **教材内容** | [ChinaTextbook](https://github.com/TapXWorld/ChinaTextbook) | 全部小初高PDF教材、教辅、名著 | 开源 ✅ |
| **苏格拉底对话** | [socratic-ai-prompt-skill](https://github.com/roy-reshef/socratic-ai-prompt-skill) | 苏格拉底式提问引导框架 | MIT ✅ |
| **知识图谱构建** | [Bookgraph](https://github.com/sumant1122/bookgraph) | 书籍概念提取 + Neo4j图谱 | 参考 |
| **音频TTS** | 硅基流动 / Fish Audio | 每日音频摘要、播客式回顾 | API |
| **LLM** | GLM-5 / DeepSeek | 对话、拆解、出题 | API |

### 我们做什么（不可替代的部分）

1. **K12教学设计层** — 带读计划、理解检测、中考考点标注（Pigou的一线教学经验）
2. **家长端** — 进度看板、理解报告、讨论指引、周报
3. **结构化阅读流程** — 不是自由聊天，是有计划的精读引导
4. **测评系统** — 对标中考题型，检测真实理解深度

### 为什么选 DeepTutor？

- ✅ Agent-native 架构，**插件体系**（Tools + Capabilities 两层插件模型）
- ✅ 内置 RAG 管道（支持多种 embedding + 向量数据库）
- ✅ 知识图谱可视化（Chart.js/SVG）
- ✅ **完整中文支持**（i18n）
- ✅ Docker 一键部署
- ✅ 活跃开发（2026年4月还在更新，v1.1.0）
- ✅ Apache-2.0 许可，商用友好
- ✅ Next.js + Python 技术栈

## 三、开发计划（缝合怪版，更快）

### Phase 0：基座搭建（3天）

- [ ] Fork DeepTutor，本地 Docker 跑起来
- [ ] 接入 GLM-5 / DeepSeek 作为 LLM
- [ ] 导入 ChinaTextbook 中的中考名著 PDF
- [ ] 跑通 RAG 流程：上传PDF → 提取文本 → 向量化 → 对话

**里程碑：能和一本名著对话**

### Phase 1：核心插件开发（2周）

**插件1：拆书引擎**
- 输入：书籍 PDF
- 输出：章节结构、人物图谱、核心知识点、中考考点标注
- 实现：DeepTutor Capabilities 插件，调用 LLM 结构化提取

**插件2：带读引擎**
- 输入：学生年级、阅读速度、时间安排
- 输出：个性化阅读计划（每天读什么、重点、讨论题）
- 实现：基于 Pigou 的教学设计，prompt 模板化

**插件3：苏格拉底对话**
- 参考 socratic-ai-prompt-skill 的框架
- 改造为中文K12语境
- 核心原则：不直接给答案，通过提问引导思考

**里程碑：完成3章《骆驼祥子》的带读demo**

### Phase 2：家长端 + 测评（2周）

**插件4：家长报告引擎**
- 阅读进度追踪
- 理解度报告（基于测评结果）
- 每周自动生成周报
- 讨论指引："今晚你可以和孩子聊..."

**插件5：测评系统**
- 每章后自动出题（选择+简答）
- 对标中考题型
- 理解度评分 + 薄弱点分析

**插件6：音频伴侣**
- 每日3-5分钟播客式回顾
- 家长周报音频版
- 接入硅基流动 TTS

**里程碑：完整带读流程 + 家长能看到报告**

### Phase 3：前端 + 多书扩展（2周）

- [ ] 微信小程序（学生端 + 家长端）
- [ ] 3本中考名著完整拆解（《骆驼祥子》《西游记》《朝花夕拾》）
- [ ] 知识图谱跨书关联
- [ ] 找5-10个学生测试

### Phase 4：打磨 + 上线（2周）

- [ ] 12本中考名著全覆盖
- [ ] 性能优化（缓存、并发）
- [ ] 收费接入（月卡39元）
- [ ] 家长社群功能（周报分享裂变）

**总计：约7-8周（对比从零开发的12-16周）**

## 六、风险与对策

| 风险 | 对策 |
|------|------|
| DeepTutor 改不动 | 它是插件架构，我们写插件不碰核心代码；最坏情况也只是用它的RAG管道 |
| ChinaTextbook 版权 | 不提供全文，只做引导+讨论+知识提取；教材本身是教育部的 |
| AI拆书不准 | Pigou 人工审核必读名著的核心内容 |
| 学生不用 | 游戏化 + 家长推动 + 中考刚需 |
| DeepTutor 更新冲突 | Fork 版本管理，我们的改动在插件层，核心可以合并上游 |
